The Kearns--Vazirani active learner (discrimination-tree variant)

Where L* maintains a flat set of representatives Q and a flat set of distinguishing suffixes T, the Kearns--Vazirani learner stores its knowledge in a binary discrimination tree: internal nodes carry a discriminator (a distinguishing suffix), and leaves carry an access string (a state representative).
A string is classified by sifting it down the tree: at each internal node with discriminator e, we ask the membership query member (u ++ e) and descend left when it holds and right otherwise; the leaf reached names the state of u.
The two developments share their underlying theory. We import Lstar.v for the Teacher interface, finite, and list_with_proof, and we reuse the membership-profile counting argument (NoDup_boollist_length from ListLemmas.v) to bound the number of states by 2|disc|.

From lstar Require Import Language DFA ListLemmas Lstar.
From Stdlib Require Import List.
From Stdlib Require Import Lia.
From Stdlib Require Import PeanoNat.
From Stdlib Require Import Setoids.Setoid.

Discrimination trees


A discrimination tree is a binary tree whose internal nodes hold a discriminator suffix and whose leaves hold an access string.
Inductive dtree : Type :=
| Leaf (access : string)
| Node (discrim : string) (lt rt : dtree).

Sifting classifies a string u by descending the tree: at Node e lt rt we go into lt when member (u ++ e) holds and into rt otherwise. The result is the access string of the leaf reached.
Fixpoint sift (t : dtree) (u : string) : string :=
    match t with
    | Leaf q ⇒ q
    | Node e lt rt ⇒
        if member (u ++ e) then sift lt u else sift rt u
    end.

The access strings (the states) of a tree.
Fixpoint leaves (t : dtree) : list string :=
    match t with
    | Leaf q ⇒ [q]
    | Node _ lt rt ⇒ leaves lt ++ leaves rt
    end.

The discriminators (the experiments) of a tree.
Fixpoint discriminators (t : dtree) : list string :=
    match t with
    | Leaf _ ⇒ []
    | Node e lt rt ⇒ e :: discriminators lt ++ discriminators rt
    end.

Sifting always lands on a leaf of the tree. This is the discrimination-tree counterpart of closed in Lstar.v: it is what makes the hypothesis transition function total, with no separate well-definedness obligation.
Lemma sift_in_leaves : ∀ t u, In (sift t u) (leaves t).

A tree always has at least one leaf, so the hypothesis always has an initial state.
Lemma leaves_nonempty : ∀ t, leaves t ≠ [].

The tree invariants: consistency, separation, and well-formedness


A tree is *consistent* when every access string sifts to its own leaf. This is the discrimination-tree counterpart of the requirement that each representative in Q be T-equivalent to itself under the transition function.
Definition consistent (t : dtree) : Prop :=
    ∀ q, In q (leaves t) → sift t q = q.

A tree is *separated* when any two distinct leaves are told apart by some discriminator of the tree. This is the discrimination-tree counterpart of separable over the discriminator set.
Definition separated (t : dtree) : Prop :=
    ∀ u v,
        In u (leaves t) → In v (leaves t) → u ≠ v →
        ∃ e, In e (discriminators t) ∧ member (u ++ e) ≠ member (v ++ e).

A tree is *well-formed* when each node's discriminator orients its two subtrees: every leaf below the true branch agrees with the discriminator, and every leaf below the false branch disagrees. This single structural invariant is the load-bearing one: NoDup (leaves t), consistent t, and separated t all follow from it (see below), so kv_learn need only carry wf.
Fixpoint wf (t : dtree) : Prop :=
    match t with
    | Leaf _ ⇒ True
    | Node e lt rt ⇒
        (∀ q, In q (leaves lt) → member (q ++ e) = true) ∧
        (∀ q, In q (leaves rt) → member (q ++ e) = false) ∧
        wf lt ∧ wf rt
    end.

Counting: the number of states is bounded by 2 ^ |discriminators|


The membership profile of a string u over the tree's discriminators: the Boolean vector recording, for each discriminator e, whether u ++ e is in the language. This is exactly the vector used in the Lstar.v counting argument loop_terminates.
Definition profile (t : dtree) (u : string) : list bool :=
    map (fun e ⇒ member (u ++ e)) (discriminators t).

Lemma profile_length : ∀ t u,
    length (profile t u) = length (discriminators t).

On a separated tree the profile is injective over the leaves: distinct leaves differ on some discriminator and hence have distinct profiles. This mirrors Hvec_inj inside Lstar.loop_terminates.
Lemma profile_injective : ∀ t,
    separated t →
    ∀ u v, In u (leaves t) → In v (leaves t) →
        profile t u = profile t v → u = v.

The number of leaves is bounded by 2|discriminators|. This is the discrimination-tree instance of the Lstar.v counting argument; the heavy lifting is done by NoDup_boollist_length from ListLemmas.v.
Lemma leaf_count_bound : ∀ t,
    NoDup (leaves t) →
    separated t →
    length (leaves t) ≤ Nat.pow 2 (length (discriminators t)).

The hypothesis DFA induced by a tree


The concrete DFA built from a tree, mirroring Lstar.make_dfa. Its states are the leaves of the tree (as a sigma type), its initial state is the leaf that nil sifts to, and its transition reads off the leaf that the one-step extension sifts to. In each case sift_in_leaves discharges the obligation that the result is a genuine state.
Definition make_dfa_kv (t : dtree) : DFA.t { q | In q (leaves t) }.
Defined.

Counterexample analysis: which leaf to split, and with which suffix


kv_p t w i is the access string of the state that the hypothesis induced by t reaches after reading the length-i prefix of w. These mirror Lstar.p and Lstar.correct.
Definition kv_p (t : dtree) (w : string) (i : nat) : string :=
    proj1_sig (run (make_dfa_kv t) (firstn i w)).

A prefix of length i is *correct* when its continuation classifies as in the language exactly when w does.
Definition kv_correct (t : dtree) (w : string) (i : nat) : Prop :=
    member (kv_p t w i ++ skipn i w) = member w.

Lemma kv_correct_dec : ∀ t w i, {kv_correct t w i} + {~ kv_correct t w i}.

The empty prefix is correct, because nil sifts to a leaf whose access string is nil (consistency), so it agrees with w on the whole word.
Lemma kv_eps_correct : ∀ t w,
    In nil (leaves t) → consistent t → kv_correct t w 0.

The full prefix is not correct, because w is a counterexample: the hypothesis accepts w exactly when member of the final leaf agrees with w, which by assumption it does not.
Lemma kv_full_not_correct : ∀ t w,
    accept_string (make_dfa_kv t) w ≠ member w →
    ¬ kv_correct t w (length w).

Splitting a leaf: replace the leaf whose access string is target by an internal node discriminating on e, keeping the original access string on one side and the fresh access string q_new on the other. Orientation follows the membership of target on e, so that target stays on the side consistent with its own profile.
Fixpoint split_leaf (t : dtree) (target e q_new : string) : dtree :=
    match t with
    | Leaf q ⇒
        if str_eq q target
        then if member (q ++ e)
             then Node e (Leaf q) (Leaf q_new)
             else Node e (Leaf q_new) (Leaf q)
        else Leaf q
    | Node e' lt rt ⇒
        Node e' (split_leaf lt target e q_new) (split_leaf rt target e q_new)
    end.

Old leaves are never removed, and the only new leaf is q_new. This direction needs no assumption about target.
Lemma split_leaves_fwd : ∀ t target e q_new x,
    In x (leaves (split_leaf t target e q_new)) →
    x = q_new ∨ In x (leaves t).

A split touches nothing when target is not a leaf.
Lemma split_leaf_id : ∀ t target e q_new,
    ¬ In target (leaves t) →
    split_leaf t target e q_new = t.

Every old leaf is preserved by a split.
Lemma split_leaves_pres : ∀ t target e q_new x,
    In x (leaves t) → In x (leaves (split_leaf t target e q_new)).

The fresh leaf appears whenever target really was a leaf.
Lemma split_q_new_in : ∀ t target e q_new,
    In target (leaves t) → In q_new (leaves (split_leaf t target e q_new)).

The converse of split_leaves_fwd, given that target is a leaf.
Lemma split_leaves_bwd : ∀ t target e q_new x,
    In target (leaves t) →
    (x = q_new ∨ In x (leaves t)) →
    In x (leaves (split_leaf t target e q_new)).

The discriminators after a split are the old ones, plus the new one e.
Lemma split_discriminators_incl : ∀ t target e q_new,
    incl (discriminators t) (discriminators (split_leaf t target e q_new)).

NoDup over concatenation


Small self-contained facts about NoDup and ++, used to reason about the leaves of Nodes.

Lemma NoDup_app_intro : ∀ (l1 l2 : list string),
    NoDup l1 → NoDup l2 → (∀ x, In x l1 → ¬ In x l2) → NoDup (l1 ++ l2).

Lemma NoDup_app_l : ∀ {A} (l1 l2 : list A), NoDup (l1 ++ l2) → NoDup l1.

Lemma NoDup_app_r : ∀ {A} (l1 l2 : list A), NoDup (l1 ++ l2) → NoDup l2.

Lemma NoDup_app_disj : ∀ {A} (l1 l2 : list A),
    NoDup (l1 ++ l2) → ∀ x, In x l1 → ¬ In x l2.

A well-oriented split preserves distinctness of leaves: target occurs once and q_new is fresh, so replacing the former by the pair adds no duplicates.
Lemma split_NoDup : ∀ t target e q_new,
    NoDup (leaves t) → In target (leaves t) → ¬ In q_new (leaves t) →
    NoDup (leaves (split_leaf t target e q_new)).

wf implies the other invariants


From well-formedness, the leaves are pairwise distinct: the two subtrees of a node are disjoint because their leaves disagree on the node's discriminator.
Lemma wf_NoDup : ∀ t, wf t → NoDup (leaves t).

From well-formedness, every access string sifts to its own leaf: at each node it descends into the subtree the discriminator orients it toward.
Lemma wf_consistent : ∀ t, wf t → consistent t.

From well-formedness, distinct leaves are separated by the discriminator at the node where their paths diverge: leaves in different subtrees disagree on that node's discriminator, and leaves in the same subtree are handled inductively.
Lemma wf_separated : ∀ t, wf t → separated t.

The converse direction we need: with distinct leaves, consistency forces well-formedness. At each node a leaf below the true branch must agree with the discriminator (else it would sift into the other subtree and, by distinctness, contradict consistency), and dually for the false branch.
Lemma consistent_NoDup_wf : ∀ t,
    NoDup (leaves t) → consistent t → wf t.

The central preservation lemma: a well-oriented split of a well-formed tree is again well-formed. The split needs only that q_new sifts to target (so it follows the same path) and that the new discriminator e tells them apart. Disjointness of the subtrees comes from wf itself, so neither NoDup nor separated is needed as a hypothesis. This subsumes both the NoDup- and the separated-preservation arguments.
Lemma split_preserves_wf : ∀ t target e q_new,
    wf t →
    In target (leaves t) →
    ¬ In q_new (leaves t) →
    sift t q_new = target →
    member (target ++ e) ≠ member (q_new ++ e) →
    wf (split_leaf t target e q_new).

Existence of a split point


Given a tree whose hypothesis mispredicts on a counterexample w, we can find a leaf target and a fresh discriminator e such that splitting target on e yields a still well-formed tree (with the empty string still a leaf, ready for the next round). This is the discrimination-tree analogue of Lstar.find_separable, and its proof follows the same shape: locate the breakpoint where the hypothesis stops tracking w, and read off the new discriminator from the remaining suffix.
Theorem find_split :
    ∀ (t : dtree) (w : string)
           (Heps : In nil (leaves t))
           (Hcons : consistent t)
           (HND : NoDup (leaves t))
           (Hce : accept_string (make_dfa_kv t) w ≠ member w),
    { target : string &
    { e : string &
    { q_new : string |
        In target (leaves t) ∧
        ¬ In q_new (leaves t) ∧
        member (target ++ e) ≠ member (q_new ++ e) ∧
        let t' := split_leaf t target e q_new in
        wf t' ∧ In nil (leaves t') }}}.

Each counterexample adds exactly one state. This is the progress guarantee for the learning loop: it never spins on a counterexample.
Lemma split_leaf_count : ∀ t target e q_new,
    NoDup (leaves t) → In target (leaves t) → ¬ In q_new (leaves t) →
    length (leaves (split_leaf t target e q_new)) = S (length (leaves t)).

The top-level learner


Unlike Lstar.lstar_opt, the Kearns--Vazirani learner needs no inner closing loop: each counterexample adds exactly one state (see split_leaf_count). The loop is therefore structurally recursive in the fuel, just as lstar_opt is, and on success the returned DFA carries a proof that it encodes L, obtained from equiv_query_correct.
Convergence within a fuel bound is not proved, as it would require the unobservable size of the minimal target DFA --- the same status as the outer loop of lstar_opt.
Fixpoint kv_learn (fuel : nat) (t : dtree)
                  (Hwf : wf t) (Heps : In nil (leaves t))
    : result { St : Type & {d : DFA.t St | encodes d} }
             { St : Type & {d : DFA.t St | True} }.
Defined.

The learner is seeded with the single-leaf tree for the empty string, which is trivially well-formed and contains nil.
Definition kv_run (fuel : nat)
    : result { St : Type & {d : DFA.t St | encodes d} }
             { St : Type & {d : DFA.t St | True} } :=
    kv_learn fuel (Leaf nil) I (or_introl eq_refl).

End KV.
